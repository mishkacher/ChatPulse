**ChatPulse** — нативная утилита для строки меню macOS, которая следит за выбранными чатами ChatGPT и автоматически отправляет команду:

> продолжай и не останавливайся до технического лимита

Приложение использует собственный встроенный браузер на системном движке **WebKit**. Google Chrome, Playwright, Ollama, платные API и отдельный ИИ-агент не нужны.

![Обзор интерфейса ChatPulse](docs/chatpulse-overview.svg)

## Статус проекта

Текущая версия: **0.2.0 alpha**.

- исходники опубликованы в основной ветке `main`;
- сборка и тесты выполняются в GitHub Actions на macOS 15;
- 20 автоматических проверок качества включены в CI;
- перед постоянным использованием требуется ручной приёмочный тест с реальным аккаунтом ChatGPT.

## Как выглядит приложение

В строке меню macOS находится значок ChatPulse. По нажатию открывается компактное меню:

- статус работы;
- одна команда **«Запустить / Остановить»**;
- ручная проверка;
- настройка интервала;
- открытие встроенного браузера;
- добавление текущего чата;
- список сохранённых чатов;
- журнал последних действий.

Отдельное окно **«Браузер ChatPulse»** похоже на упрощённый Safari: кнопки назад, вперёд, обновить, открыть ChatGPT, строка текущего адреса и кнопка **«Добавить чат»**.

## Что реализовано

- встроенный `WKWebView` на WebKit;
- постоянное локальное хранение cookies и данных входа;
- отдельное видимое окно для входа и выбора чатов;
- отдельный скрытый WebKit-контроллер для фоновых проверок;
- сохранение названий, URL и состояния чатов;
- интервалы 1, 2, 5, 10, 15 и 30 минут;
- собственный интервал от 30 секунд;
- **at-most-once** защита: один ответ не получает команду повторно даже при неопределённом DOM-подтверждении после клика;
- ожидание завершения ответа;
- пассивная первая проверка после запуска;
- подтверждение появления отправленного сообщения, когда интерфейс позволяет это проверить;
- сохранение пользовательских изменений даже во время долгой проверки;
- полностью русский пользовательский интерфейс и журнал;
- модульные тесты и автоматический аудит.

## Что удалено намеренно

Детектор технического лимита отсутствует. ChatPulse не анализирует баннеры или тексты лимитов и не пытается обходить ограничения ChatGPT.

Если поле ввода или кнопка отправки недоступны, команда не форсируется. Ошибка записывается в журнал, а следующая попытка происходит только при очередной плановой проверке.

## Главная логика

1. На первой проверке ChatPulse запоминает текущий ответ и ничего не отправляет.
2. При появлении нового ответа приложение снова только запоминает его.
3. Если на следующей проверке ответ не изменился, завершён и принадлежит ассистенту, отправляется команда продолжения.
4. До клика ошибки считаются безопасными для повторной попытки на следующем интервале.
5. После фактического клика отпечаток ответа помечается обработанным независимо от того, успел ли DOM подтвердить появление сообщения.
6. Один и тот же ответ не может получить команду дважды.
7. Перед отправкой повторно проверяется, что чат не был удалён или отключён пользователем.

## Требования

- macOS 13 Ventura или новее;
- подключение к интернету;
- аккаунт ChatGPT;
- Swift 6 / Xcode Command Line Tools только для сборки из исходников.

## Установка из исходников

```bash
git clone https://github.com/mishkacher/ChatPulse.git
cd ChatPulse
bash scripts/install_app.sh
```

Приложение устанавливается в `/Applications/ChatPulse.app`, а без прав записи — в `~/Applications/ChatPulse.app`.

## Первое использование

1. Запустите ChatPulse.
2. Нажмите значок приложения в строке меню.
3. Выберите **«Открыть браузер ChatPulse…»**.
4. Войдите в ChatGPT внутри встроенного окна.
5. Откройте нужный чат.
6. Нажмите **«Добавить чат»**.
7. Повторите для остальных чатов.
8. Выберите интервал и нажмите **«Запустить»**.

## Cookies и вход

Встроенный браузер использует постоянное хранилище WebKit. Авторизация сохраняется между запусками ChatPulse, но не связана с cookies, историей или вкладками Safari.

### Ограничение входа через Google

Google может блокировать OAuth-авторизацию внутри встроенных WebView. В таком случае используйте другой доступный способ входа в ChatGPT, например электронную почту, пароль, код, passkey или вход через Apple.

## Хранение настроек

```text
~/Library/Application Support/ChatPulse/settings.json
```

В настройках сохраняются:

- названия и URL чатов;
- включённое состояние;
- интервал проверки;
- отпечатки последних ответов;
- время последней отправки;
- результат подтверждения последней отправки.

Содержимое всей переписки в файл настроек не записывается.

## Проверка проекта

```bash
make test
make audit
make build
```

`make audit` запускает модульные тесты и 20 обязательных проверок проекта.

## Ограничения alpha-версии

- интерфейс ChatGPT может измениться и потребовать обновления селекторов;
- вход и отправка должны быть проверены на реальном аккаунте;
- сборка подписывается локальной ad-hoc подписью, а не Apple Developer ID;
- приложение не обходит и не ускоряет сброс лимитов ChatGPT.
