# Релизный чек-лист ChatPulse

## Автоматические ворота

Каждый релизный commit обязан пройти пять независимых macOS 15 циклов. В каждом цикле выполняются:

- `swift test`;
- `swift test -c release`;
- 20 обязательных quality gates;
- синтаксическая проверка shell-скриптов;
- release-сборка `ChatPulse.app`;
- проверка версии и номера сборки из `VERSION` и `BUILD_NUMBER`;
- `plutil -lint` для `Info.plist`;
- строгая проверка code signature;
- проверка bundle identifier, иконки и режима menu-bar приложения.

После успешной CI основной ветки workflow `Release`:

1. повторно выполняет релизный preflight в чистом macOS-окружении;
2. собирает ZIP-архив;
3. создаёт SHA-256 checksum;
4. создаёт тег `v<version>`;
5. публикует GitHub Release и оба файла.

## Ручной приёмочный тест

Перед длительным использованием на реальном аккаунте:

1. скачать релизный ZIP и проверить checksum;
2. переместить `ChatPulse.app` в Applications;
3. подтвердить первый запуск через **Открыть**, если Gatekeeper показывает предупреждение;
4. открыть встроенный браузер;
5. пройти вход по доступному способу;
6. добавить тестовый чат;
7. убедиться, что первая проверка ничего не отправляет;
8. убедиться, что стабильный завершённый ответ получает ровно одну команду на следующей проверке;
9. получить новый ответ и убедиться, что команда не отправляется немедленно;
10. убедиться, что следующая неизменившаяся проверка отправляет ровно одну команду;
11. проверить остановку мониторинга во время проверки;
12. проверить оба скина и сохранение выбора после перезапуска;
13. проверить сохранение списка чатов и интервала;
14. проверить удаление приложения скриптом `scripts/uninstall_app.sh`.

## Подписание и нотарификация

Текущий открытый релиз использует ad-hoc подпись. Для распространения без предупреждения Gatekeeper необходимы:

- активная подписка Apple Developer Program;
- сертификат **Developer ID Application**;
- профиль keychain / CI secrets;
- hardened runtime;
- отправка архива в Apple Notary Service;
- stapling нотариального ticket к приложению;
- повторная проверка `spctl --assess`.

Нельзя объявлять сборку нотарифицированной, пока все эти этапы фактически не завершены.
