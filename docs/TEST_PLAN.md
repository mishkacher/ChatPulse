# План тестирования

## Пять независимых CI-циклов

Каждый Pull Request запускает матрицу из пяти отдельных macOS 15 runner. Каждый runner начинает с чистого checkout и выполняет полный набор:

1. проверку формата `VERSION` и `BUILD_NUMBER`;
2. `swift test`;
3. `swift test -c release`;
4. синтаксическую проверку всех shell-скриптов;
5. 20 автоматических quality gates;
6. release-сборку `ChatPulse.app`;
7. проверку `Info.plist`, bundle identifier, версии и build number;
8. проверку иконки;
9. строгую проверку code signature и hardened runtime.

`fail-fast` отключён, поэтому результат каждого цикла виден независимо. Пятый успешный цикл формирует ZIP и SHA-256 checksum.

## Автоматическая публикация

После успешной CI основной ветки workflow `Release`:

1. получает проверенный commit;
2. повторно выполняет `scripts/release_preflight.sh`;
3. собирает приложение в чистом macOS-окружении;
4. формирует `ChatPulse-macOS-v<version>.zip`;
5. создаёт и проверяет SHA-256;
6. создаёт тег `v<version>`;
7. публикует GitHub Release и оба файла;
8. не публикует ту же версию повторно.

## Автоматические тесты логики

Проверяются:

- пассивная первая проверка;
- ожидание одного интервала после нового ответа;
- отправка для стабильного завершённого ответа;
- запрет повторной отправки;
- at-most-once поведение после неподтверждённого клика;
- ожидание сообщения ассистента;
- блокировка во время генерации;
- обработка ошибки страницы;
- сохранение состояния в JSON;
- атомарная запись настроек;
- нормализация URL;
- распознавание URL Google OAuth без ложных срабатываний;
- официальный URL входа ChatGPT;
- подготовка поля email без чтения или экспорта данных;
- поиск официальной команды `Try with email`;
- проверка WebAuthn через `PublicKeyCredential`;
- отсутствие самостоятельного создания passkey;
- распознавание завершённого входа;
- наличие ровно двух скинов;
- стабильные raw value скинов;
- Codable round-trip модели скина;
- точные цвета темы ChatPulse Preview;
- сохранение выбора под ключом `ChatPulse.ui.skin`;
- границы интервала;
- вместимость журнала;
- отсутствие детектора технического лимита;
- отсутствие зависимости от Chrome;
- контролируемые WebKit-попапы;
- ограничение окон видимой областью экрана;
- установка без зависимости от executable-бита скриптов;
- воспроизводимость версии из `VERSION` и `BUILD_NUMBER`;
- наличие release notes, changelog, preflight и release workflow;
- отсутствие внешнего ИИ и платного API.

## Ручной приёмочный тест macOS

### Установка

1. Скачать ZIP и checksum из GitHub Release.
2. Выполнить `shasum -a 256 -c` и получить `OK`.
3. Переместить приложение в Applications.
4. Подтвердить первый запуск стандартным способом Gatekeeper, не отключая его полностью.
5. Открыть **«О ChatPulse…»** и проверить версию и build number.

### Интерфейс и скины

6. Открыть встроенный браузер.
7. Проверить переключатель скина в браузере и меню строки состояния.
8. Выбрать **macOS** и проверить системные материалы и светлый/тёмный режим.
9. Выбрать **ChatPulse Preview** и проверить градиент, текст, контролы и кнопку добавления.
10. Изменить размер окна и проверить геометрию темы.
11. Перезапустить приложение и проверить сохранение скина.
12. Убедиться, что тема сайта ChatGPT не изменена приложением.

### Авторизация

13. Проверить наличие меню **«Войти ▾»** и строки состояния.
14. Пройти доступный email-сценарий с самым новым одноразовым кодом.
15. Проверить пункт повторного запроса email-кода на поддерживаемом шаге.
16. Проверить passkey при наличии зарегистрированного ключа.
17. Проверить безопасное сообщение, когда passkey отсутствует.
18. Нажать Google и убедиться, что основной браузер не заменяется `about:blank`.
19. Убедиться, что дочернее окно не превышает видимую область экрана.
20. Перезапустить ChatPulse и проверить сохранение WebKit-сессии.

### Мониторинг

21. Добавить два тестовых чата.
22. Установить интервал одну минуту.
23. Убедиться, что первая проверка ничего не отправляет.
24. Убедиться, что стабильный ответ получает одну команду на следующей проверке.
25. Убедиться, что повторная команда не появляется.
26. Получить новый ответ и проверить новый цикл ожидания.
27. Остановить мониторинг во время проверки и убедиться, что отправка отменена.
28. Отключить или удалить чат во время проверки и убедиться, что отправка отменена.
29. Перезапустить приложение и проверить сохранение чатов и интервала.
30. Убедиться, что скрытый браузер не переключает видимое окно.

## Нотарификация

Ad-hoc релиз не должен ошибочно проходить как нотарифицированный. После добавления Apple Developer ID отдельный приёмочный набор обязан включать:

- проверку Developer ID-подписи;
- `spctl --assess --type execute`;
- отправку в Apple Notary Service;
- stapling ticket;
- повторную проверку Gatekeeper на чистом Mac.
